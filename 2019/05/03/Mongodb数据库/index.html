Mongodb数据库

发表于 2019-05-03 | 更新于 2019-05-08 | 分类于 Mongodb

Mongodb

概念

  • mongod 启动数据库服务（服务，最小化）
  • mongo 连接数据库(shell)

数据库（database）

  • 存放数据的仓库，可以存放集合

集合（collection）

  • 类似数组，可以存放文档

文档（document）

  • 数据的最小单位，我们存储和操作的内容都是文档

    如果进入的数据库和集合不存在，在第一次创建所属文档时自动创建

基础命令

  1. show dbs：显示所有数据库

  2. use dbName：进入某数据库

  3. db：表示当前数据库对象
  4. show collections：显示数据库中的所有集合

CRUD

提前将所有需要更改的数据准备好再一次性操作数据库可以大幅度提升性能（减少数据库操作）

插入

  • db.collectionName.insert(data)：在当前数据库，某集合下插入data对象

    • 当 data 为数组的时候，可同时插入多个对象

    • 如果 data 对象中没有 _id属性，数据库会自动创建 _id（必须唯一）

查询

  • db.collectionName.find(查询对象)： 查询某集合下的所有文档

    • 不传查询对象时：查询所有文档
    • {查询字段：值}：返回该集合下某字段（key）值（value）相同的文档
    • find 方法返回的是一个数组，[索引] 可以取得其中某一个
    • .count()：返回结果的数量（.length()）
    • 查询内嵌文档时，查询字段必须加引号 {"obj.xxx": 123}
    • 查询对象之后还能再传入一个过滤对象，{key: 1}：查询结果只显示 key属性 的投影

  • db.collectionName.findOne()：查询第一个结果（返回一个文档）

修改

  • db.collectionName.update(查询对象, 新对象)

    • 会覆盖替换
    • 默认只操作第一个结果

  • db.collectionName.update(查询对象, {$set: 修改对象})

    • 修改操作符，不替换

  • db.collectionName.update(查询对象, {$unset: 移除对象})

  • db.collectionName.updateMany()：操作查询到的所有结果

  • db.collectionName.update(查询对象, {$push: data})：向数组中添加 data（允许重复）
  • db.collectionName.update(查询对象, {$addToSet: data})：向数组中添加 data（不能重复）

删除

  • db.collectionName.remove(查询对象, [boolean])：删除文档

    • 不写第二个参数时，默认删除所有结果
    • 第二个参数为 true，只删除第一个查询结果
    • 必传查询对象，{}：删除所有文档（性能较差）

  • db.collectionName.drop()：删除集合

  • db.dropDatabase()：删除数据库

操作符

  1. $lt：key < value

  2. $lte：key <= value

  3. $gt：key > value

  4. $gte：key >= value

  5. $ne：key != value

  6. $eq：key === value

  7. $or：查询对象 || 查询对象

  8. $inc：key.val += x （自增）

方法

  1. .limit(num)：只查询 n 条数据
  2. .skip(num)：跳过前 n 条数据

分页： db.collectionName.find(data).skip(20).limit(10) 跳过前 20 条结果后，再显示 10 条结果

  1. .sort({key: 1})：结果排序，key 表示要排序的字段，值 1 升序 -1 降序，多个 key 依次排序

文档间的关系

一对一

  • 内嵌文档（父文档对象 内嵌套 子文档对象）

一对多

  • 内嵌文档
  • 设置 子文档对象 的 user_id 等于 父文档对象 的 _id

多对多

  • 设置 子文档对象 的 users_id 为一个 数组， 里面包含多个父文档对象 的 _id

Mongoose

对象文档模型（ODM）库，通过 node 来操作数据库的模块

  1. 为文档创建约束（Schema）
  2. 对模型中的对象 / 文档进行验证
  3. 数据类型转换为对象
  4. 使用中间件来和业务逻辑挂钩
  • Schema（模式对象）

定义约束了数据库中的文档结构

  • Model

作为集合中的所有文档的表示，相当于collection

  • Document

具体的文档

安装

  1. npm i mongoose --save：安装

  2. const mongoose = require('mongoose')：引入

  3. mongoose.connect('mongodb://服务器地址:端口/数据库名', {useMongoClient: true})

    不写端口号默认 27017

  4. mongoose 对象中 connection 属性表示数据库连接，可以为其绑定事件监听

  5. mongoose.disconnect()：断开数据库连接

    非关系型数据库一般不需要主动断开连接

使用

Model CRUD

  • myModel.count(查询对象, [回调函数])：查询 Model 集合中符合条件的 Document 对象的数量

新增

  1. myModel.create(docs, [callBack])：创建一个（文档对象）或多个（数组）文档添加到数据库中

查询

  1. myModel.find(查询条件, [投影], [查询选项], 回调函数)：返回查询结果数组
  2. myModel.findById(id, [投影], [查询选项], 回调函数)：根据 id 字符串查询，返回具体对象
  3. myModel.findOne(查询条件, [投影], [查询选项], 回调函数)：只查第一个，返回具体对象

在回调函数里取结果，参数 1 错误对象， 参数 2，docs（数组）

投影可以用对象配置：key 为需要获取的字段名，val 1 为显示 0 为不显示

也可以传一个字符串：以 空格 分割多个字段名，-字段名 表示不显示

修改

  1. myModel.update(查询对象, 修改对象, [配置参数], [回调函数])

  2. myModel.updateOne(查询对象, 修改对象, [配置参数], [回调函数])

  3. myModel.updateMany(查询对象, 修改对象, [配置参数], [回调函数])
  4. myModel.replaceOne(查询对象, 修改对象, [配置参数], [回调函数])

删除

  1. myModel.remove(查询对象, [回调函数])
  2. myModel.deleteOne(查询对象, [回调函数])
  3. myModel.deleteMany(查询对象, [回调函数])

Document

  • 文档对象和数据库中的文档一一对应
  • 它是对应 Model 集合的实例
