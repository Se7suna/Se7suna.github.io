ES6

ECMA 和严格模式

ECMA

  1. 它是一种由 ECMA 组织（前身为欧洲计算机制造商协会）制定和发布的脚本语言规范
  2. 我们学的 JavaScript 是 ECMA 的实现, 但术语 ECMAScript 和 JavaScript 平时表达同一个意思

  3. JS 包含三个部分：

    • ECMAScript（核心）
    • 扩展 ==> 浏览器端
      1. BOM（浏览器对象模型）
      2. DOM（文档对象模型）
    • 扩展 ==> 服务器端

  4. ES 的几个重要版本

    • ES5: 09 年发布
* ES6 (ES2015): 15年发布, 也称为ECMA2015

* ES7 (ES2016): 16年发布, 也称为ECMA2016 (变化不大)

严格模式

  1. 理解

    • 除了正常运行模式(混杂模式)，ES5 添加了第二种运行模式: “严格模式” (strict mode)
    • 顾名思义，这种模式使得 Javascript 在更严格的语法条件下运行

  2. 目的/作用

    • 消除 Javascript 语法的一些不合理、不严谨之处，减少一些怪异行为
    • 消除代码运行的一些不安全之处，为代码的安全运行保驾护航
    • 为未来新版本的 Javascript 做好铺垫

  3. 使用

    • 在全局或函数的第一条语句定义为: 'use strict'
    • 如果浏览器不支持, 只解析为一条简单的语句, 没有任何副作用

  4. 语法和行为改变

    • 必须用var声明变量

    • 禁止自定义的函数中的this指向window

    • 创建eval作用域

    • 对象不能有重名的属性

Object 拓展

ES5 给 Object 扩展了一些静态方法, 常用的 2 个

  1. Object.create(新创建对象的原型对象, [可选: 新建对象自身属性描述符])

    • 作用: 以指定对象为原型创建新对象作为返回值返回

    • 可以为新的对象指定自身的属性(一个或多个属性都需要封装成对象传入)

    • 对每个属性的数据属性进行描述(描述也需要封装成对象,包含以下属性)

      value: 属性的默认值

      writable: 属性值是否是可修改 默认为false

      configurable: 属性是否可重新定义 默认为false

      enumerable: 属性是否能用for in枚举 默认为false

  2. Object.defineProperties(obj, prop, descriptor)

    • 作用: 为指定对象精确添加或修改对象的属性(属性描述符封装成对象传入)

      1. 数据描述符:

        configurable: 该属性是否可重新定义

        writable: 该属性值是否是可修改

        value: 该属性的默认值

        enumerable: 该属性是否能用 for in 枚举

      2. 访问描述符:

        设置扩展属性的值的回调函数, 在使用扩展属性的时候才调用, 返回动态计算出的该属性的值(惰性求值)

        修改扩展属性的值时触发的回调函数, 修改后的值作为实参传入

  3. delete obj.xxx 删除 obj 对象上的 xxx 属性

  4. 返回对象中的所有数据：

    • let arr = Object.keys(obj)

    • let arr = Object.values(obj)

    • let arr = Object.entries(obj)

      结果数组 [ [key, value], … ]

面试题: 区别bind()与call()和apply()?

  1. 都能指定函数中的this

Let 和 Const

let 关键字

  1. 作用:

    与var类似, 用于声明一个变量

  2. 特点:

    • 在块作用域内有效 (ES6 中添加)
    • 不能重复声明同名变量
    • 不会预处理, 既不存在变量提升

  3. 应用:

    • 循环遍历加监听
    • 使用let取代var是趋势

  4. 暂时性死区

    • 在 let 定义变量之前的区域都无法使用该变量名

const 关键字

  1. 作用: 定义一个常量 (不可改变引用)

  2. 特点:

    • 不能修改

    对象可以修改它的属性, 但是不能重新赋值引用

    • 其它特点同let

  3. 应用: 保存不用改变的数据

箭头函数

  • 作用:

    方便定义匿名函数, 调整this指向

    1. (形参) => {函数内容}
    2. 没有形参时, 需要用括号占位
    3. 一个参数时, 括号可以省略
    4. 函数体只有一条语句时, 可以不用大括号, 默认return语句执行后的结果
    5. 函数体如果有多条语句: 需要用{}包裹, 若有需要返回的内容需要设置return
    6. 箭头函数不能做构造函数和 Generator 函数

  • 常用场景:

    回调函数

  • 注意

    1. 箭头函数没有 arguments
    2. 箭头函数不能 new

  • this

    1. 箭头函数没有自己的this

      • 它的this不是调用时决定的
      • 它的this指向箭头函数定义时所属的作用域

    2. 理解:

- 箭头函数的`this`看它的外层是否有其他函数
- 如果有, 箭头函数的`this`和外层函数相同
- 如果没有, 则指向 `window`

Promise

promise 主要用于处理异步请求

  • 语法

  • 注意点

    1. Promise 构造函数执行时立即调用传入的函数

    2. resolve 和 reject 两个回调函数作为参数

    3. 调用resolve()时修改状态为成功 fulfilled

    4. 调用reject()时修改状态为失败 rejected

    5. then()最多可传入两个回调函数, 对应异步成功和失败

    6. then()的返回值仍然是个 promise 对象（默认成功状态）

      如果需要返回失败的 pormise 对象，需要 return Promise.reject()

    7. catch()捕获错误会直接跳过 失败 promise 后的所有then

      使用catch捕获错误，该then就不需要传入失败的回调

      catch 默认也是返回 成功的 promise

  • Promise.all(arr)：必须所有异步执行成功才调用 resolve，返回结果数组
  • Promise.race()：只要有一个异步操作完成就调用相应的回调
  • Promise.prototype.finally()：所有异步执行完成后进行回调（无论成功失败）

Symbol

概念: ES6 中添加的一种原始数据类型 Symbol（它是原始数据类型，所以不能 new）

已有的 6 种原始数据类型：String, Number, Boolean, null, undefined, Object

作用:

  • 作为私有属性的变量名，保护私有变量

特点:

  1. Symbol 表示一个唯一的值，解决命名冲突问题
  2. Symbol 的值不能与其他数据进行计算，包括同字符串拼串
  3. for in, for of遍历时不会遍历 symbol属性
  4. 不能做任何运算，不能转换成数字

使用:

  1. 调用 Symbol 函数返回一个 Symbol

    注意: 不能使用 new

  2. symbol 作为属性名时必须使用属性选择器 []

Iterator

  • 概念: iterator 是一种接口机制, 为各种不同的数据结构提供遍历机制

  • 作用:

    1. 为各种数据结构, 提供一个统一的、简便的遍历接口
    2. 使得数据结构的成员能够按某种次序排列
    3. ES6 创造了一种新的遍历命令for...of循环，iterator 接口主要供for...of使用

  • 工作原理:

    1. 创建一个指针对象(遍历器对象), 指向数据结构的起始位置

    2. 第一次调用next()方法, 指针自动指向数据结构的第一个成员

    3. 接下来不断调用next()方法, 指针会一直往后移动, 直到指向最后一个成员

    4. 每次调用next()方法返回的是一个包含value和done的对象

      {value: 当前成员的值, done: 布尔值}

      value 表示当前成员的值，done 对应的布尔值表示当前的数据的结构是否遍历结束

      当遍历结束的时候返回的 value 值是 undefined，done 值为 false

  • 原生具备 iterator 接口的数据:
    1. Array
    2. arguments
    3. String
    4. set
    5. map

当使用 for of 去遍历某一个数据结构时

  1. 首先在目标上找Symbol.iterator
  2. 找到则可以用for of遍历 否则报错
  3. ...运算符 和 解构赋值 默认也是调用 iterator 接口

Generator

  • 概念

    1. 生成器函数在执行时能暂停, 后面又能从暂停处继续执行

    2. 调用一个生成器函数并不会马上执行它里面的语句, 而是返回一个这个生成器的迭代器/遍历器(iterator)对象

    3. 当这个遍历器的next()方法被调用时, 其内的语句会执行到出现yield的位置为止

      包括紧跟 yield 的语句

    4. yield后紧跟遍历器要返回的值

    5. next()方法返回一个对象, 这个对象包含两个属性: value和done

      value属性表示本次yield表达式的返回值

      done属性为布尔类型, 表示生成器后续是否还有yield语句, 即生成器函数是否已经执行完毕并返回

    6. 调用next()方法时，如果传入了参数, 那么这个参数会作为上一条执行的yield的值（用一个变量保存，可以在之后的语句中使用这个值）

  • 用途

    1. ES6 提供的解决异步编程的方案之一
    2. Generator 函数是一个状态机，内部封装了不同状态的数据
    3. 用来生成遍历器对象

  • 特点

    1. function 与函数名之间有一个星号
    2. 内部用yield表达式来定义不同的状态
    3. 可暂停函数(惰性求值)

  • 注意

    1. generator 函数返回的是指针对象(iterator)，而不会执行函数内部逻辑
    2. 调用next()方法函数内部逻辑开始执行，遇到yield表达式停止，返回{value: yield后的表达式结果/undefined, done: false/true}
    3. 再次调用next()方法会从上一次停止时的yield处开始, 到下一个yield为止
  1. yield语句返回结果通常为undefined, 当调用next()方法时传参内容会作为启动时yield的返回值

`class`声明创建一个基于原型继承的具有给定名称的新类. 类声明**不允许**再次声明已经存在的类，否则将会抛出一个类型错误。
  1. 通过class定义类/实现类的继承
  2. 在类中通过constructor定义构造方法
  3. 通过new来创建类的实例
  4. 通过extends来实现类的继承
  5. 通过super调用父类的构造方法
  6. 可在子类中重写父类中继承的一般方法
  7. 类声明不可以提升
  8. 方法必须使用简写

数组

  2. Array.of(v1, v2, v3): 将一系列值转换成数组

  3. arr.find(function(value, index, arr){return true}): 返回第一个满足条件返回 true 的元素

  • 浅拷贝数据: 拷贝引用, 操作会影响原数据
  1. 直接赋值给另一个变量

  3. Array.prototype.concat()

Set 和 Map 容器

set

  • 类似数组 无序 不可重复, 可存储任何数据类型, 无论是原始值或者是对象

语法: let mySet = new Set(arr)

  1. add(value): Set 对象尾部添加一个元素, 返回该 set对象

  2. delete(value): 移除 set 中与这个值相等的元素，返回操作结果的布尔值

  3. has(value): 返回一个布尔值, 表示该值在 Set 中存在与否

  4. clear(): 清空 set 对象

  5. size: set 中储存的数据的数量

6) 可以用for...of进行遍历 mySet / mySet.keys() / mySet.values()

7) 数组去重

map

  • 类似对象 有序 不可重复, 用于保存键值对, 任何值(对象或者原始值) 都可以作为一个键或一个值

for...of 循环为每次迭代返回一个[key，value]数组

和 Object 的区别:

  1. Map 的键可以是任意值, 包括函数、对象
  2. Map 中的键值是有 序的 (添加时的顺序)
  3. size 属性直接获取一个 Map 的键值对个数
  4. Map 是可迭代的

语法: let myMap = new Map([ [key1, value1], [key2, value2]... ])

  1. set(key, value): 像Map中添加数据, 返回该 map对象
  2. get(key): 返回键对应的值，如果不存在则返回 undefined
  3. delete(key): 移除任何与键相关联的值，并且返回操作结果的布尔值
  4. has(key): 返回一个布尔值，检测该 map 中是否包含某个 key
  5. clear(): 移除 map对象 中的所有数据
  6. size: 返回 map 的键值对个数

for…of

  1. 遍历数组
  2. 遍历 Set
  3. 遍历 Map
  4. 遍历字符串
  5. 遍历伪数组

for...of / for...in / forEach 的区别:

  • for...in

    1. index索引为字符串型数字, 不能直接进行几何运算

    2. 遍历顺序有可能不是按照实际的内部顺序

    3. 使用for...in会遍历数组所有的可枚举属性, 包括原型上的

      适合遍历对象 index ==> key

  • for...of

  1. 遍历的只是数组内的元素, 而不包括数组的原型

  2. 不能遍历普通对象, 因为没有 iterator 接口

    适合遍历数组 index ==> value

  • forEach(callback(Value, index, arr))
    1. 对数组的每个元素执行一次提供的回调函数
    2. 只能遍历数组
    3. 无法break或者return中断

async 函数

async 函数

  1. 当调用一个async函数时, 会返回一个Promise对象
  2. 当这个async函数返回一个值时, Promise的resolve()方法会负责传递这个值
  3. 当async函数抛出异常时, Promise的reject()方法也会传递这个异常值
  4. async函数中可能会有await表达式, 可以使async函数暂停执行
  5. 等表达式中的Promise解析完成后继续执行async函数并返回解决结果

await 表达式

  1. 会暂停当前async函数的执行, 等待Promise处理完成
  2. 若Promise正常处理(fulfilled), 其回调的resolve函数参数作为await表达式的值，继续执行async函数
  3. 若Promise处理异常(rejected), await表达式会把Promise的异常原因抛出
  4. 如果await操作符后的表达式的值不是一个Promise, 那么该值将被转换为一个已正常处理的Promise

注意: await关键字仅仅在async function中有效

概念: 真正意义上去解决异步回调的问题，同步流程表达异步操作

本质: Generator的语法糖

特点:

  1. 不需要像 Generator 去调用next()方法, 遇到await等待, 当前的异步操作完成就往下执行
  2. 返回的总是Promise对象, 可以用then()方法进行下一步操作
  3. async取代 Generator 函数的星号*, await取代 Generator 的yield
  4. 语意上更为明确, 使用简单, 暂时没有任何副作用
